Static Website Crawler

This EDS block allows you to crawl any website and extract all internal and external URLs. Perfect for site audits, link analysis, and SEO purposes.

Domain
https://main--gattex--onetakeda.aem.live/

Features

  • Comprehensive Discovery: Finds URLs from HTML content, sitemaps, and structured data
  • Smart Filtering: Separates internal pages from external links
  • CORS Handling: Uses fallback proxy services when direct access is blocked
  • Export Capabilities: Download results as text or JSON files
  • Rate Limiting: Respectful crawling with configurable delays
  • Error Tracking: Comprehensive reporting of failed requests

How to Use

To use this block in your EDS project:

  1. Copy the static-website folder from blocks/ to your project
  2. Add the block to your page using the standard EDS block format
  3. Configure the domain URL you want to crawl
  4. Click "Start Crawling" to begin the discovery process

Block Configuration

You can configure the crawler by adding rows to the block:

  • Domain: The website URL to crawl (required)
  • URL: Alternative parameter name for the domain

Technical Implementation

The crawler works by:

  1. Starting with the provided domain URL
  2. Attempting to fetch and parse sitemap.xml files
  3. Crawling each discovered page to extract all links
  4. Following internal links recursively (with a safety limit)
  5. Categorizing discovered URLs as internal or external
  6. Providing comprehensive export options

Note: Due to browser CORS policies, some websites may not be crawlable directly. The block includes fallback mechanisms using proxy services.
