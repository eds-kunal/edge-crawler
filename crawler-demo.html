Website Crawler Demo

This demo shows how to use the static website crawler block to crawl and discover all URLs from a website.

How to Use

The block can be configured by providing a domain URL. Here are some examples:

Example 1: Default Configuration

The block will use the default domain if no configuration is provided:

Domain
https://main--gattex--onetakeda.aem.live/

Example 2: Custom Domain

You can specify a different domain to crawl:

URL
https://www.adobe.com

Features

  • Comprehensive Crawling: Discovers URLs from HTML content, sitemaps, and various link sources
  • CORS Handling: Uses proxy services when direct requests are blocked by CORS
  • Internal vs External: Separates internal pages from external links
  • Export Functionality: Download results as text or JSON files
  • Rate Limiting: Respectful crawling with delays between requests
  • Error Tracking: Reports errors and failed requests
  • Real-time Progress: Shows crawling progress and statistics

Technical Details

The crawler works by:

  1. Starting with the provided domain URL
  2. Attempting to fetch and parse the sitemap.xml
  3. Crawling each discovered page and extracting all links
  4. Following internal links recursively (up to a limit)
  5. Categorizing links as internal or external
  6. Providing export options for the discovered URLs

Note: Due to browser security restrictions (CORS), some websites may not be crawlable directly. The block attempts to use proxy services as a fallback.
